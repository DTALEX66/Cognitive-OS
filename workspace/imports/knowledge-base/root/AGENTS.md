# Workspace Safety Guard — 全盘安全策略

## 磁盘布局规则
| 盘符 | 用途 | 保护级别 |
|------|------|----------|
| C: | 系统盘 | 🔴 禁止任何删除/卸载/修改操作 |
| D: | 软件盘 | 🟡 禁止工作区外删除，禁止卸载任何软件 |
| E: | **资料/数据盘** | 🔴 **绝对禁止任何删除/修改操作** |
| F: | 娱乐盘 | 🟡 禁止工作区外删除 |

## 不可违反的安全规则（CODE — 四盘边界保护）

### 规则 1：E 盘（资料/数据盘）— 绝对禁区
- **禁止** 对 E:\ 及其任何子目录执行 Remove-Item、del、m、mdir、Move-Item 操作。
- **禁止** 修改 E 盘上的任何文件元数据、权限、属性。
- **禁止** 对 E 盘执行 Format-Volume、磁盘清理、压缩等操作。
- E 盘是只读保护模式——**只允许读取，不允许写入或删除**。

### 规则 2：C 盘（系统盘）— 系统保护区
- **禁止** 删除 C 盘上任何文件/文件夹。
- **禁止** 卸载 C 盘上任何已安装的软件。
- **禁止** 修改系统环境变量、注册表、服务、用户、组、防火墙、计划任务。

### 规则 3：D 盘（软件盘）— 受限区
- **禁止** 删除 D:\ 根目录及任何非工作区路径的文件。
- **禁止** 卸载 D 盘上任何已安装的软件。
- **允许** 在工作区 D:\Project Directory\KnowledgeBase\* 内正常读写。

### 规则 4：F 盘（娱乐盘）— 受限区
- **禁止** 删除 F:\ 根目录及任何非工作区路径的文件。

### 规则 5：操作前必须验证
在调用任何 Remove-Item、Uninstall、del、m、Format-Volume 之前，**必须先调用** .codex\guard\uninstall-guard.ps1 -TargetPath <目标路径> -OperationType Delete 进行安全检查。

---

**违反上述规则的后果：2.4TB 数据永久丢失。必须严格遵守。**
