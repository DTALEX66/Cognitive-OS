# 07 DECISIONS — 架构决策记录

## D-001: 选择SQLite而非PostgreSQL
- **决策**: 使用 SQLite + FTS5
- **原因**: 本地优先，零配置，单文件便携
- **日期**: 2026-06

## D-002: 选择FastAPI而非Django
- **决策**: 使用 FastAPI
- **原因**: 轻量、异步、自动OpenAPI文档
- **日期**: 2026-06

## D-003: A/B双线架构
- **决策**: A线(人类学习)与B线(机器知识)分离
- **原因**: 不同目标、不同评估体系
- **日期**: 2026-06

## D-004: v0.2 MVP收敛
- **决策**: 从大而全收敛到本地知识库+轻量学习+轻量机器辅助
- **原因**: 先跑通核心闭环，再扩展
- **日期**: 2026-06-24

## D-005: Frozen目录策略
- **决策**: 实验代码代码保留在backend/，索引放在frozen/
- **原因**: 不破坏导入，随时可恢复
- **日期**: 2026-06-24
