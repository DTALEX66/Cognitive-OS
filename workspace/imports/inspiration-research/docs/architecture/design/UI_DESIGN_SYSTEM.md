# UI 设计规范

## 1. 设计目标

Lumina AI 的 UI 必须表达：准确、安静、可信、轻盈、有诗意、可长期使用。它不是游戏 UI，也不是普通后台管理系统。

## 2. 视觉关键词

```text
光感
星图
路线
锚点
记忆资产
知识校准
安静科技感
本地可信感
```

## 3. 布局规则

| 区域 | 规范 |
|---|---|
| 顶部导航 | 保持清晰，优先可理解：首页 / 路线 / 社区 / 报告 / 设置 |
| 左侧流程栏 | 展示核心流程：上传、复习、宫殿、训练、报告 |
| 主内容区 | 卡片式结构，标题 + 数据 + 操作清晰分层 |
| 右侧面板 | 用于摘要、状态、质量评分、行动建议 |
| Footer | 显示运行状态、版本号、协议入口 |

## 4. 色彩原则

- 主色：蓝紫渐变，用于 AI、路线、主按钮。
- 辅助色：绿色用于通过、可信、本地运行；橙色用于提醒；红色只用于错误和风险。
- 背景：白色 / 浅灰 / 极浅蓝紫，不使用大面积深色。
- 卡片：圆角、轻阴影、边框透明度低，避免厚重。

## 5. 字体与文案层级

| 层级 | 用法 |
|---|---|
| H1 | 页面诗意标题，如“知途星库” |
| H2 | 模块标题，如“路线资产” |
| Plain title | 功能解释，如“我的路线” |
| Subtitle | 说明产品价值 |
| Badge | 状态与可信标识 |
| Button | 操作必须清楚，不宜过度诗意 |

## 6. 组件规范

- Button：主按钮只用于下一步、创建、确认、导出等高优先级行为。
- Card：每张卡必须有明确标题、内容和操作区。
- Badge：状态可视化，不替代解释文本。
- QualityBar：只展示 0-100 可量化指标。
- RouteNode：必须展示事实层和记忆层的关联。

## 7. 响应式规范

当前优先桌面端 1440px。后续补：
- 1024px 平板布局。
- 390px 手机复习/训练布局。
- 侧边栏折叠。

## 8. 禁止事项

```text
过度霓虹
游戏化大场景抢占学习主线
按钮全诗意化导致用户看不懂
随机图片破坏记忆稳定
不同页面组件风格不统一
```
