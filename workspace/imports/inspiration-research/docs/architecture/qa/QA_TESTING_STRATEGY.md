# QA 与测试策略

## 1. 静态测试

必须运行：

```bash
npm run prehandoff
git diff --check
```

已覆盖：
- 源码编码与空白字符。
- i18n 字典 key 对齐。
- RoutePackage 基础结构。
- UI 参考图和缩略图资产。
- 路线工具存在性。
- 本地学习闭环页面存在性。
- 规范文件存在性。

## 2. 工程测试

依赖安装后运行：

```bash
npm run lint
npm run typecheck
npm run test
npm run build
```

## 3. 页面手工验收

| 页面 | 验收点 |
|---|---|
| `/` | 首页品牌、CTA、五大引擎、双语切换 |
| `/routes` | Demo 路线加载、导入、导出、本地保存 |
| `/upload` | 上传区域、解析预览、下一步入口 |
| `/review` | 节点切换、事实层/记忆层、确认节点 |
| `/palace` | 路线节点、冻结状态、视觉映射 |
| `/training` | 答题、提示、查看答案、结果记录 |
| `/report` | 读取训练记录并生成统计 |
| `/community` | 路线卡片、审核标签、改动摘要 |
| `/settings` | 双语、质量门禁说明 |

## 4. 回归重点

- 刷新页面后本地路线不丢。
- 冻结路线不自动变化。
- 训练结果影响报告。
- 双语切换刷新后保持。
- 导入错误 JSON 有提示。
- 所有图片路径有效。
