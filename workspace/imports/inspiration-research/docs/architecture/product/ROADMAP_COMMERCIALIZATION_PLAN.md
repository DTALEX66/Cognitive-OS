# 路线图与商业化规划

## Stage 0：当前静态商用 MVP

目标：完成高质感 UI、静态流程、本地路线包、质量门禁和规范体系。

验收：
- UI 参考图完整。
- 所有核心页面可访问。
- i18n key 对齐。
- demo route package 有效。
- 静态门禁通过。

## Stage 1：工程可运行版本

目标：在真实开发环境中通过 `npm run lint/typecheck/test/build`。

补齐：
- package-lock.json。
- ESLint / TypeScript 细节修复。
- 页面级手动回归。
- 组件拆分和复用。

## Stage 2：本地资产稳定版

目标：用户可长期保存路线、训练记录和版本资产。

补齐：
- IndexedDB。
- 图片 Blob / 静态资产引用。
- 路线版本历史。
- 导入导出 ZIP。

## Stage 3：AI 记忆编码版

目标：接入本地/云端 AI，生成记忆场景脚本和候选图。

约束：
- AI 只生成候选。
- 用户确认后冻结。
- 每个视觉元素必须映射到事实关键词。

## Stage 4：社区路线可信版

目标：发布、调用、分叉、审核和差异对比。

约束：
- 事实层改动强提醒。
- 分享模板不包含个人训练数据。
- 社区排序不只看热度。

## Stage 5：商用增长版

目标：付费模板、教师认证、机构空间、云端增强。

商业模式：
- 免费本地版。
- Pro 云端增强版。
- 教师/专家路线模板包。
- 机构私有路线库。
