# 文案与双语规范

## 1. 双语切换位置

右上角，搜索框右侧、通知铃铛左侧。

```text
🌐 中 / EN
```

## 2. i18n 规则

- 所有跨页面复用文案必须进入 `data/i18n/zh-CN.json` 和 `data/i18n/en-US.json`。
- 两个语言包 key 必须完全一致。
- 页面临时 Demo 文案可先硬编码，但进入商用前必须迁移到 i18n。
- 新增页面必须同步新增英文。

## 3. 诗意文案使用边界

| 类型 | 是否诗意化 |
|---|---|
| 页面主标题 | 可以 |
| 模块标题 | 可以轻度诗意 |
| 按钮 | 以清晰为主 |
| 风险提示 | 不诗意化，必须准确 |
| 空状态 | 可以温和诗意 |
| 社区审核 | 不诗意化，必须明确 |

## 4. 翻译风格

中文：含光感、路线感、记忆感，但不玄。
英文：清晰、现代 SaaS，不要直译古风词。

## 5. 新增文案流程

1. 先写普通功能文案。
2. 再写诗意标题。
3. 添加 i18n key。
4. 中英文同步。
5. 运行 `npm run validate:i18n` 或 `npm run prehandoff`。
