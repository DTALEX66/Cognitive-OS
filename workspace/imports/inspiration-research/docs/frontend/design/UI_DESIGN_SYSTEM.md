# UI Design System

## Direction

The interface should feel like a serious Apple-style productivity tool: calm surfaces, clear hierarchy, subtle glass, precise spacing, and restrained motion. It should read as technical through data density, route maps, status panels, and command surfaces rather than decorative gradients.

## Components

- App shell: persistent navigation, user status, command entry.
- Panels: flat sections for workflows, cards only for repeated items.
- Route surfaces: timeline, trust panel, review panel, training panel.
- Feedback: toast notifications for sync, save, import, and export events.
- Motion: short entrance and state transitions through `motion`.

## Constraints

- Avoid one-color themes.
- Keep cards below large marketing proportions.
- Prefer icon buttons with tooltips for tools.
- Do not ship empty panels or fake counters.
