# 06 | 学习画像系统 — Learning Profile System

## 一、系统定位：学习画像在 OS 中的角色

### 1.1 从 CC cost-tracker 到学习画像

A-Line Human Learning OS 的学习画像系统，脱胎于 Codex CLI 的 cost-tracker 模块（302 行）。在 Codex CLI 中，cost-tracker 负责追踪 LLM API 调用的 Token 消耗和费用，为系统提供成本核算和资源管理能力。它的核心函数包括：

- **getStoredSessionCosts()**：读取已存储的会话费用历史
- **saveCurrentSessionCosts()**：保存当前会话的费用快照
- **formatTotalCost()**：将原始费用数据格式化为可读报表
- **addToTotalSessionCost()**：累加当前会话的增量费用

将这个机制迁移到学习领域，我们得到的是一个**完整的学习画像系统**——它将原始的学习行为数据（时间投入、完成次数、正确率、知识点分布等）转化为结构化的学习者画像，类似于 cost-tracker 将原始 Token 消耗转化为费用报表。

核心迁移逻辑：

| CC cost-tracker | A-Line 学习画像 | 迁移说明 |
|----------------|----------------|---------|
| Token 消耗追踪 | 学习投入追踪 | 追踪每个知识点的学习时间、练习次数 |
| 会话费用保存 | 每日学习记录 | 保存每天的学习画像快照 |
| 总费用格式化 | 学习统计报表 | 生成可读的学习统计和趋势分析 |
| 增量费用累加 | 复习次数累加 | 累加正确率、掌握度等指标 |

### 1.2 在 A-Line 架构中的位置

学习画像系统处于 A-Line 的**评估层与规划层之间**，是连接过去学习表现与未来学习计划的桥梁：

```
输入层（Capture 子系统）
    ↓
诊断层（Diagnose 子系统）→ 定位当前知识状态
    ↓
反馈闭环（05 子系统）→ 生成每一次学习的反馈数据
    ↓
★★★ 学习画像系统（Learning Profile）★★★
    ↓
    ├──→ 趋势分析：发现学习模式的变化
    ├──→ 弱点识别：定位需要加强的领域
    ├──→ 效果评估：衡量学习策略的有效性
    └──→ 画像输出：为路径规划提供输入
            ↓
规划层（Plan 子系统）→ 根据画像调整学习路径
```

### 1.3 设计原则

- **自动化采集**：学习行为数据自动记录，无需手动输入
- **多维度分析**：从时间、正确率、速度、知识点分布等多维度构建画像
- **可视化呈现**：画像以直观的图表和报表形式呈现
- **隐私安全**：画像数据本地存储，不自动上传
- **可追溯**：每个画像指标都能追溯到原始学习记录

### 2.1 核心数据模型

```python
from dataclasses import dataclass
from datetime import date, datetime
from typing import Dict, List, Optional

@dataclass
class LearningRecord:
    """单次学习行为的原始记录"""
    id: str                          # 记录唯一标识
    timestamp: datetime              # 发生时间
    topic: str                       # 知识点/主题
    sub_topic: str                   # 子知识点
    task_type: str                   # 任务类型（做题/复习/阅读/实践）
    duration_seconds: int            # 耗时（秒）
    is_correct: Optional[bool]       # 是否正确（可空，阅读类无此字段）
    confidence: Optional[int]        # 自评信心（1-5，可空）
    difficulty: float                # 任务难度（0.0-1.0）

@dataclass
class DailyProfile:
    """每日学习画像快照"""
    date: date                       # 日期
    total_time_minutes: int          # 总学习时间（分钟）
    total_tasks: int                 # 总任务数
    correct_count: int               # 正确数
    accuracy: float                  # 正确率
    subjects: List[str]              # 涉及的学科列表
    topics: Dict[str, TopicStats]    # 各知识点的统计
    new_topics_learned: int          # 新学知识点数量
    reviewed_topics: int             # 复习知识点数量
    weak_points: List[str]           # 薄弱点列表
    strong_points: List[str]         # 优势点列表
    energy_level: Optional[float]    # 精力水平（可空，来自主观评估）

@dataclass
class TopicStats:
    """单个知识点的统计"""
    attempts: int                    # 尝试次数
    correct: int                     # 正确次数
    accuracy: float                  # 正确率
    total_time_minutes: int          # 总投入时间
    last_review_date: date           # 最后复习日期
    mastery_level: float             # 掌握度（0.0-1.0）
    streak: int                      # 连续正确次数
```

## 三、成本核算映射 — 将 API Token 消耗映射为学习投入

### 3.1 映射表

CC cost-tracker 追踪的是 Token 消耗，A-Line 学习画像系统追踪的是"学习投入"——将 Token 的概念替换为学习领域中的量化指标：

| CC 概念 | A-Line 学习概念 | 计算方式 |
|---------|----------------|---------|
| Token 数量 | 学习次数 | 每次学习行为 = 1 次投入 |
| Prompt Token | 主动学习时间 | 学习者主动投入的时间（做题、阅读、思考） |
| Completion Token | 被动学习时间 | 接收反馈、查看答案解析的时间 |
| 总 Token 消耗 | 总学习投入 | 时间 + 次数 + 精力消耗的综合指标 |
| 每会话 Token | 每次学习会话 | 一次持续学习的所有行为聚合 |
| Token 价格 | 学习效率系数 | 时间投入与学习产出的比值 |

### 3.2 学习投入核算函数

```python
class LearningCostTracker:
    """类比 CC 的 cost-tracker，核算学习投入"""

    def calculate_session_cost(self, session_records: List[LearningRecord]) -> SessionCost:
        """核算一次学习会话的总投入"""
        total_tasks = len(session_records)
        total_time = sum(r.duration_seconds for r in session_records)
        correct_count = sum(1 for r in session_records if r.is_correct)
        difficulty_sum = sum(r.difficulty for r in session_records)

        # 学习投入综合得分（类比 Token 消耗）
        effort_score = (
            total_tasks * 1.0 +
            total_time / 60 * 2.0 +
            difficulty_sum * 3.0
        )

        return SessionCost(
            session_id=session_records[0].session_id,
            total_tasks=total_tasks,
            total_time_seconds=total_time,
            correct_count=correct_count,
            accuracy=correct_count / total_tasks if total_tasks > 0 else 0,
            avg_difficulty=difficulty_sum / total_tasks if total_tasks > 0 else 0,
            effort_score=effort_score
        )

    def save_session_cost(self, cost: SessionCost, daily_profile: DailyProfile):
        """保存本次会话的学习投入到当日画像（类比 saveCurrentSessionCosts）"""
        # 累加到当日画像
        daily_profile.total_tasks += cost.total_tasks
        daily_profile.total_time_minutes += cost.total_time_seconds // 60
        daily_profile.correct_count += cost.correct_count
        # 重新计算正确率
        daily_profile.accuracy = (
            daily_profile.correct_count / daily_profile.total_tasks
            if daily_profile.total_tasks > 0 else 0
        )
        # 保存到文件
        self._write_daily_profile(daily_profile)

    def format_total_effort(self, profiles: List[DailyProfile]) -> str:
        """格式化总学习投入（类比 formatTotalCost）"""
        total_time = sum(p.total_time_minutes for p in profiles)
        total_tasks = sum(p.total_tasks for p in profiles)
        avg_accuracy = (
            sum(p.correct_count for p in profiles) / total_tasks
            if total_tasks > 0 else 0
        )
        return (
            f"总学习时间: {total_time // 60}小时{total_time % 60}分钟\n"
            f"总练习次数: {total_tasks}次\n"
            f"综合正确率: {avg_accuracy * 100:.1f}%\n"
            f"总学习天数: {len(profiles)}天"
        )

    def add_to_total_session_effort(self, current_total: EffortTotal, new_cost: SessionCost):
        """累加到总会话投入（类比 addToTotalSessionCost）"""
        current_total.total_tasks += new_cost.total_tasks
        current_total.total_time_seconds += new_cost.total_time_seconds
        current_total.total_correct += new_cost.correct_count
        current_total.total_effort_score += new_cost.effort_score
```

### 3.3 效率分析

```python
class EfficiencyAnalyzer:
    def analyze_efficiency(self, profiles: List[DailyProfile]) -> EfficiencyReport:
        # 时间效率：每小时的正确题目数
        recent = profiles[-7:]
        total_correct = sum(p.correct_count for p in recent)
        total_hours = sum(p.total_time_minutes for p in recent) / 60
        tasks_per_hour = total_correct / total_hours if total_hours > 0 else 0

        # 效率趋势
        week1 = profiles[-14:-7] if len(profiles) >= 14 else profiles
        week2 = profiles[-7:]
        trend = self.compare_efficiency(week1, week2)

        return EfficiencyReport(
            tasks_per_hour=tasks_per_hour,
            trend=trend,
            suggestion=self.make_suggestion(trend)
        )
```

## 四、画像生成 — 如何从原始数据生成学习画像

### 4.1 画像生成流水线

```
原始学习记录（LearningRecord 列表）
    ↓
[Step 1] 数据清洗
    ↓ 去重、补全缺失字段、标准化格式
[Step 2] 会话聚合
    ↓ 按时间窗口分组，形成学习会话
[Step 3] 日画像生成
    ↓ 聚合单日数据，生成 DailyProfile
[Step 4] 知识点画像更新
    ↓ 更新每个知识点的 TopicStats
[Step 5] 画像分析
    ↓ 识别弱点、优势、趋势
[Step 6] 画像持久化
    ↓ 写入 daily/ 和 topics/ 目录
```

### 4.2 画像生成器实现

```python
class ProfileGenerator:
    def generate_daily_profile(
        self, records: List[LearningRecord], date: date
    ) -> DailyProfile:
        # 基础统计
        total_tasks = len(records)
        correct = sum(1 for r in records if r.is_correct is True)
        total_time = sum(r.duration_seconds for r in records if r.duration_seconds)

        # 按知识点聚合
        topic_stats = self.aggregate_by_topic(records)

        # 识别薄弱点和优势点
        weak_points = self.identify_weak_points(topic_stats)
        strong_points = self.identify_strong_points(topic_stats)

        return DailyProfile(
            date=date,
            total_time_minutes=total_time // 60,
            total_tasks=total_tasks,
            correct_count=correct,
            accuracy=correct / total_tasks if total_tasks > 0 else 0,
            subjects=list(set(r.topic.split("/")[0] for r in records)),
            topics=topic_stats,
            new_topics_learned=...,    # 对比历史数据得出
            reviewed_topics=...,       # 对比历史数据得出
            weak_points=weak_points,
            strong_points=strong_points,
            energy_level=None,
        )

    def aggregate_by_topic(
        self, records: List[LearningRecord]
    ) -> Dict[str, TopicStats]:
        """按知识点聚合统计"""
        topic_groups = {}
        for r in records:
            if r.topic not in topic_groups:
                topic_groups[r.topic] = []
            topic_groups[r.topic].append(r)

        result = {}
        for topic, recs in topic_groups.items():
            correct = sum(1 for r in recs if r.is_correct is True)
            result[topic] = TopicStats(
                attempts=len(recs),
                correct=correct,
                accuracy=correct / len(recs) if len(recs) > 0 else 0,
                total_time_minutes=sum(r.duration_seconds for r in recs) // 60,
                last_review_date=max(r.timestamp.date() for r in recs),
                mastery_level=self.calc_mastery(topic, recs),
                streak=self.calc_streak(recs),
            )
        return result

    def calc_mastery(self, topic: str, records: List[LearningRecord]) -> float:
        """综合计算掌握度"""
        recent = records[-10:]
        if not recent:
            return 0.0
        accuracy = sum(1 for r in recent if r.is_correct) / len(recent)
        # 考虑时间衰减：越久远的记录权重越低
        weights = [1.0 / (i + 1) for i in range(len(recent))]
        weighted_accuracy = sum(
            (1 if r.is_correct else 0) * w
            for r, w in zip(recent, weights)
        ) / sum(weights)
        return weighted_accuracy
```

### 4.3 每周画像聚合

```python
class WeeklyProfileAggregator:
    def aggregate(self, daily_profiles: List[DailyProfile]) -> WeeklyProfile:
        if not daily_profiles:
            return WeeklyProfile()

        total_time = sum(d.total_time_minutes for d in daily_profiles)
        total_tasks = sum(d.total_tasks for d in daily_profiles)
        total_correct = sum(d.correct_count for d in daily_profiles)

        # 合并所有薄弱点，按出现频率排序
        all_weak = [
            point for d in daily_profiles for point in d.weak_points
        ]
        weak_freq = Counter(all_weak)

        return WeeklyProfile(
            week=self.get_week_number(daily_profiles[0].date),
            days_learned=len([d for d in daily_profiles if d.total_tasks > 0]),
            total_time_hours=total_time / 60,
            total_tasks=total_tasks,
            accuracy=total_correct / total_tasks if total_tasks > 0 else 0,
            avg_daily_time=total_time / len(daily_profiles),
            top_weak_points=weak_freq.most_common(5),
            trend=self.calc_trend(daily_profiles),
        )
```

## 五、可视化报表 — 学习统计和趋势分析

### 5.1 报表类型

学习画像系统支持以下报表类型：

**日报（Daily Report）：**
- 今日学习总览：时间、任务数、正确率
- 知识点分布：今日涉及的知识点及各自表现
- 薄弱点提醒：今日表现最差的知识点
- 连续学习天数：打卡记录

**周报（Weekly Report）：**
- 本周与上周的对比趋势
- 各知识点的掌握度变化
- 错误类型分布
- 学习效率分析（时间/正确率比值）

**月报（Monthly Report）：**
- 本月学习全景：各学科的时间分配、正确率排行
- 知识掌握度热力图
- 进步最大的知识点排行榜
- 建议下月重点攻克的方向

### 5.2 报表生成器

```python
class ReportGenerator:
    def generate_daily_report(self, profile: DailyProfile) -> str:
        """生成日报告（类比 formatTotalCost 的格式化功能）"""
        report = f"""
## 📅 学习日报 - {profile.date}

### 今日概况
- 学习时间：{profile.total_time_minutes} 分钟
- 完成任务：{profile.total_tasks} 个
- 正确率：{profile.accuracy * 100:.1f}%
- 新学知识点：{profile.new_topics_learned} 个
- 复习知识点：{profile.reviewed_topics} 个

### 知识点表现
"""
        for topic, stats in sorted(
            profile.topics.items(),
            key=lambda x: x[1].accuracy
        ):
            report += (
                f"- {topic}: 正确率 {stats.accuracy * 100:.0f}% "
                f"（{stats.correct}/{stats.attempts}）"
                f" 掌握度: {stats.mastery_level:.0%}\n"
            )

        if profile.weak_points:
            report += f"\n### ⚠️ 薄弱点\n"
            for wp in profile.weak_points[:3]:
                report += f"- {wp}\n"

        report += f"\n### 状态\n"
        report += f"- 连续学习：{profile.date}（{self.calc_streak(profile.date)}天）\n"
        return report

    def generate_weekly_trend(self, weekly: WeeklyProfile) -> str:
        """生成周趋势分析"""
        trend_icon = "📈" if weekly.trend == "improving" else "📉"
        return f"""
## {trend_icon} 周报 - 第{weekly.week}周

### 总体数据
- 学习天数：{weekly.days_learned}/7 天
- 总学习时间：{weekly.total_time_hours:.1f} 小时
- 总任务数：{weekly.total_tasks} 个
- 综合正确率：{weekly.accuracy * 100:.1f}%
- 日均学习：{weekly.avg_daily_time:.0f} 分钟

### 主要薄弱点
{self.format_weak_points(weekly.top_weak_points)}

### 趋势
{self.format_trend_description(weekly.trend)}
"""
```

### 5.3 关键指标追踪

画像系统持续追踪以下核心指标，并在报表中可视化：

- **掌握度曲线**：每个知识点在时间轴上的掌握度变化，反映学习效果
- **正确率趋势**：日/周正确率的移动平均线，过滤短期波动
- **学习速度**：单位时间内的学习产出（正确题数/小时）
- **遗忘曲线拟合**：基于 FSRS 参数的个性化遗忘曲线
- **投入产出比**：学习时间与正确率提升的比值

## 六、与反馈闭环的联动 — 反馈数据如何影响画像

### 6.1 反馈数据的画像注入

反馈闭环（05 子系统）产生的每一次反馈数据，都会流入学画像系统并影响画像：

```python
class ProfileUpdater:
    """从反馈数据更新学习画像"""

    def feed(self, feedback_records: List[FeedbackRecord]):
        """接收反馈闭环的输出并更新画像"""
        for record in feedback_records:
            # 1. 更新当日画像
            date = record.timestamp.date()
            daily = self.load_or_create_daily(date)

            # 更新基础统计
            daily.total_tasks += 1
            if record.correct:
                daily.correct_count += 1

            # 更新知识点统计
            if record.topic not in daily.topics:
                daily.topics[record.topic] = TopicStats(...)
            ts = daily.topics[record.topic]
            ts.attempts += 1
            if record.correct:
                ts.correct += 1
            ts.accuracy = ts.correct / ts.attempts
            ts.total_time_minutes += record.time_spent // 60
            ts.last_review_date = date
            ts.mastery_level = self.recalc_mastery(record.topic, ts)

            # 2. 根据错误类型更新薄弱点列表
            if not record.correct:
                self.update_weak_points(daily, record)

            # 3. 根据评估等级更新掌握度趋势
            self.update_mastery_trend(record.topic, ts.mastery_level, date)

            # 4. 保存更新后的画像
            self.save_daily(daily, date)

    def update_weak_points(self, daily: DailyProfile, feedback: FeedbackRecord):
        """更新薄弱点列表"""
        topic = feedback.topic
        # P0-P2 级别的错误，加入薄弱点
        if feedback.error_category in (ErrorCategory.P0, ErrorCategory.P1, ErrorCategory.P2):
            if topic not in daily.weak_points:
                daily.weak_points.append(topic)
        # 如果后续表现好，从薄弱点移除
        elif feedback.correct and topic in daily.weak_points:
            daily.weak_points.remove(topic)
```

### 6.2 反馈驱动的画像指标更新

不同的反馈类型驱动不同的画像指标更新：

| 反馈类型 | 画像指标 | 更新逻辑 |
|---------|---------|---------|
| 正确完成（S级） | mastery_level + 0.15 | 大幅提升掌握度 |
| 正确完成（A级） | mastery_level + 0.08 | 中等提升掌握度 |
| P0 级错误 | mastery_level - 0.25 | 大幅降低掌握度，加入薄弱点 |
| P1 级错误 | mastery_level - 0.15 | 降低掌握度，加入薄弱点 |
| P2 级错误 | mastery_level - 0.10 | 方法性错误的专项标记 |
| P4/P5 级错误 | mastery_level - 0.03 | 微调，不加入薄弱点 |
| 连续 3 次正确 | streak_bonus | 触发掌握度加速提升机制 |

### 6.3 反馈数据的画像聚合

画像系统定期聚合反馈闭环的历史数据，生成更高层次的洞察：

```python
class FeedbackAggregator:
    def aggregate_to_profile(self, all_feedback: List[FeedbackRecord]) -> ProfileInsights:
        insights = ProfileInsights()

        # 1. 计算每个知识点的综合掌握度
        for topic in self.get_all_topics(all_feedback):
            topic_feedback = [f for f in all_feedback if f.topic == topic]
            insights.topic_mastery[topic] = self.calc_mastery(topic_feedback)

        # 2. 识别薄弱知识点集群
        insights.weak_clusters = self.find_weak_clusters(
            insights.topic_mastery,
            self.get_knowledge_graph()  # 从知识宫殿获取知识图谱
        )

        # 3. 计算遗忘速度
        insights.forgetting_rate = self.calc_forgetting_rate(all_feedback)

        # 4. 推荐最佳复习间隔
        insights.recommended_intervals = self.recommend_intervals(
            insights.forgetting_rate
        )

        return insights
```

## 七、画像对路径规划的影响 — 画像如何调整学习计划

### 7.1 画像驱动的路径调整

学习画像直接影响路径规划（03 子系统）的决策：

```python
class ProfileDrivenPlanner:
    """根据学习画像调整学习计划"""

    def generate_plan(self, profile: LearningProfile) -> WeeklyPlan:
        plan = WeeklyPlan()

        # 1. 根据薄弱点安排复习
        for weak_topic, mastery in profile.weak_topics:
            if mastery < 0.4:
                plan.add_task(ReviewTask(
                    topic=weak_topic,
                    priority="high",
                    suggested_exercises=5,
                    review_mode="intensive"
                ))
            elif mastery < 0.7:
                plan.add_task(ReviewTask(
                    topic=weak_topic,
                    priority="medium",
                    suggested_exercises=3,
                    review_mode="normal"
                ))

        # 2. 根据优势点安排进阶
        for strong_topic, mastery in profile.strong_topics:
            if mastery > 0.85:
                plan.add_task(AdvanceTask(
                    topic=self.find_next_level(strong_topic),
                    priority="medium"
                ))

        # 3. 根据学习速度调整任务量
        efficiency = profile.efficiency.tasks_per_hour
        if efficiency > 5:      # 效率高，增加任务量
            plan.task_multiplier = 1.2
        elif efficiency < 2:    # 效率低，减少任务量
            plan.task_multiplier = 0.8

        # 4. 根据连续学习天数调整强度
        if profile.current_streak > 30:
            plan.suggest_rest_day = True   # 建议休息日

        return plan
```

### 7.3 自适应学习计划生成

画像系统积累的数据越多，生成的计划就越个性化：

```python
class AdaptivePlanner:
    def generate_personalized_plan(self, profile: LearningProfile) -> Plan:
        # 基于历史数据的个性化参数
        optimal_daily_time = self.find_optimal_time(profile)
        preferred_subject_order = self.find_preferred_order(profile)
        difficulty_curve = self.fit_difficulty_curve(profile)

        return Plan(
            daily_time=optimal_daily_time,
            subject_order=preferred_subject_order,
            difficulty_progression=difficulty_curve,
        )

    def find_optimal_time(self, profile) -> int:
        """基于历史发现最佳每日学习时长"""
        # 分析历史数据：哪段时间投入产出比最高
        best_weeks = sorted(
            profile.weekly_profiles,
            key=lambda w: w.accuracy / max(w.avg_daily_time, 1),
            reverse=True
        )[:3]
        optimal = int(
            sum(w.avg_daily_time for w in best_weeks) / len(best_weeks)
        )
        return min(optimal, 120)  # 上限2小时
```

### 7.4 画像驱动路径规划的完整循环

```
[学习行为发生]
    ↓
[反馈闭环（05）处理]
    ↓ 输出：反馈记录
[画像系统更新]
    ↓ 输出：画像快照
[画像分析引擎]
    ├── 薄弱点识别
    ├── 掌握度评估
    ├── 效率分析
    ├── 趋势发现
    └── 模式识别
            ↓
[路径规划引擎（03）]
    ├── 调整复习计划
    ├── 调整难度曲线
    ├── 调整学习内容
    └── 调整学习强度
            ↓
[下一轮学习任务]
    ↓
[反馈闭环再次处理] → 画像持续更新 → 路径持续优化
```

这个循环意味着：学习画像不是静态的快照，而是随着每一次学习行为不断演化的动态模型。画像越丰富，路径规划越精准；路径执行得越好，画像数据越有价值。这是一个持续正向迭代的过程。
