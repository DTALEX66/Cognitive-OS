## 一、系统定位

### 1.1 诊断+巩固在整个学习 OS 中的角色

诊断巩固系统位于捕获系统之后、计划系统之前，是 A-Line 学习流水线的第二站，承担着从原料中识别问题并设计解决方案的核心职能。

完整学习链路中，诊断与巩固分别扮演不同角色：

捕获 → 诊断 → 计划 → 执行 → 巩固 → 反馈

诊断是对捕获到的学习资料和学习者的知识状态进行分析，识别薄弱点、盲区和效率瓶颈。巩固是基于诊断结果，设计间隔重复卡片、复习计划、知识整理方案等巩固活动。

两者的关系可以类比医疗场景：诊断是体检和化验，巩固是治疗方案。没有诊断的巩固是盲目的，没有巩固的诊断是无效的。

### 1.2 与 Claude Code Compact 系统的渊源

本系统的设计直接受 Claude Code 的 Compact 系统启发。Compact 系统解决的是当 AI 对话上下文窗口空间不足时，如何压缩历史消息保留关键信息的问题。这套机制在 8 个文件、3454 行代码中实现了 5 种不同的压缩策略。

迁移到学习系统语境后，原系统的概念发生如下映射：

- 上下文窗口 → 学习者的认知容量 / 注意力带宽
- Token 计数 → 知识量 / 学习负荷的量化
- 摘要 (summary) → 知识浓缩 / 核心笔记
- 压缩策略 → 诊断 / 巩固方法
- 触发阈值 → 诊断触发阈值
- Forked Agent 压缩 → 异步巩固引擎
- PreCompact/PostCompact Hooks → 诊断前/巩固后处理

### 1.3 两大模块的交互模式

诊断和巩固虽然是同一个子系统中的两个侧面，但时间粒度和复杂度不同。诊断负责分析状态与识别问题，巩固负责生成解决方案和复习材料。巩固的结果会回馈到诊断系统，形成闭环迭代。


## 二、5 种压缩/巩固策略详解

以下五种策略从 Claude Code Compact 系统的五种压缩策略演化而来，每一种都对应一个具体的学习场景。

### 策略 1：全量巩固（对应 CC compactConversation）

适用场景：月度深度回顾 / 学期综合复习 / 考试前系统复习

流程：触发 → 收集该周期内所有学习记录 → 知识图谱重建 → 生成全量巩固包 → 校验完整性 → 交付

1. 全量盘点：收集指定周期内所有捕获的资料、笔记、错题
2. 知识图谱重建：提取关键概念及其关系，构建该科目/主题的知识网络
3. 生成全量巩固包：包括核心概念速查表、易错点清单、概念关系图
4. 完整性校验：检查是否覆盖了该周期计划的全部学习目标

优点：最全面，不遗漏任何知识点。知识图谱重建有助于发现概念间的隐藏联系。适合考试前或项目里程碑时的系统复习。

缺点：开销最大，耗时最长。不适合日常使用（频率建议每月1次或学科切换时1次）。

与 CC Compact 的差异：CC 的全量压缩是替换全部消息为摘要；A 线全量巩固不是在已有知识库基础上追加一次高密度的结构化复习包。

### 策略 2：增量巩固（对应 CC partialCompactConversation）

适用场景：每周学习回顾 / 新增知识点的针对性复习

流程：检测上次巩固时间点 → 提取新知识点 → 只处理新增和变更部分 → 增量合并 → 更新索引

1. 变更识别：对比上次巩固时间戳后的捕获记录，筛选新资料和修改过的笔记
2. 差异化处理：只对新知识点做语义提取和卡片生成
3. 增量合并：将新生成的巩固内容嵌入已有的知识结构和复习计划

优点：效率高，只处理增量。保留已有巩固成果不变，不干扰已建立的记忆模型。适合每周例行执行。

缺点：长期依赖增量模式可能导致知识结构碎片化，需要定期做一次全量巩固来重整碎片。

### 策略 3：自动触发巩固（对应 CC autoCompact）

适用场景：后台自动监控，学习状态异常时主动介入

触发条件包括：
- 错题率超标（> 30%）：数据来自反馈系统
- 遗忘率超标（> 40% 未通过间隔复习）：数据来自 Palace 记忆系统
- 学习停滞（连续 3 天无新捕获）：数据来自捕获系统
- 知识密度过高（单日捕获 > 50 页资料）：数据来自捕获系统
- 诊断置信度降低（薄弱点识别失败率 > 20%）：数据来自诊断引擎

流程：监控线程每秒检查学习指标 → 任一指标超标 → 评估优先级 → 如果多个指标同时超标则全量巩固 → 如果单指标轻度超标则增量巩固加标记关注

优点：用户无感，后台守护。在学习出现问题早期即可介入，防止问题累积。自动适配学习节奏。

缺点：阈值设置不当可能导致过度触发或漏触发。需要一定的训练期来校准个性化参数。

### 策略 4：微巩固（对应 CC microCompact）

适用场景：每日 5 分钟快速回顾 / 临睡前快速复盘

流程：轻量触发 → 只取当日核心 3-5 个知识点 → 快速生成自测问题 → 完成即丢弃

1. 轻量采样：仅提取当天捕获资料中最核心的 3-5 个知识点
2. 快速生成：每个知识点生成 1-2 个主动回忆问题
3. 即时完成：用户在 5 分钟内完成自测，结果不存入长期索引
4. 每日重置：微巩固的内容是用完即弃的，不干扰正式巩固计划

优点：开销极小，适合每天执行。帮助形成日回顾习惯。低成本保持学习状态。

缺点：深度不足，无法处理复杂概念。不产生可复用的结构化输出。

执行模板：今日微巩固 5 分钟包括 30 秒回顾今天学了什么描述 3 个核心概念、2 分钟自测问题、30 秒标记最难的一个概念。

### 策略 5：长期记忆巩固（对应 CC sessionMemoryCompact）

适用场景：跨月/跨年的长期知识点维护 / 已掌握内容的状态更新

流程：标记已掌握概念 → 检查巩固间隔 → 对间隔将超期的概念触发复习 → 更新掌握程度

1. 掌握度评估：对每个知识点维护一个掌握度分数（0-100）
2. 间隔计算：参考 Ebbinghaus 遗忘曲线和间隔重复科学
   - 掌握度 100：30 天后需检查
   - 掌握度 80：14 天后需检查
   - 掌握度 60：7 天后需检查
   - 掌握度 40：3 天后需检查
   - 掌握度 < 40：需要重新学习
3. 巩固执行：仅对间隔将超期的知识点触发生成复习卡片
4. 状态更新：巩固后更新掌握度和下次巩固间隔

优点：专注于长期记忆维护，避免学过就忘。与 Anki / SuperMemo 等间隔重复工具天然对接。

缺点：前期需要建立掌握度数据（冷启动问题）。对已掌握的定义需要学习者主观评估。


## 三、自动触发逻辑

### 3.1 上下文监控机制

类比 CC autoCompact.ts 中的 getEffectiveContextWindowSize，学习系统中需要建立认知负荷监控器：

每日认知负荷 = 当日捕获资料复杂度 * 资料量 + 待巩固队列长度 + 错题积压量

监控指标包括：捕获认知负荷（资料页数乘难度系数，正常0-50，警戒51-80，危险大于80）、待巩固队列（未处理巩固任务数，正常0-20，警戒21-50，危险大于50）、错题积压（未复习错题数，正常0-30，警戒31-60，危险大于60）、连续学习天数（7天提醒休息，14天强制休息）、新概念密度（每周10-30正常，31-50警戒，大于50危险）。

### 3.2 阈值触发决策树

监控检查后计算综合评分，轻度超标触发快速诊断，中度超标触发增量诊断，重度超标触发全量诊断，诊断完成后自动选择巩固策略。

### 3.3 频率控制

快速诊断每小时最多1次，增量诊断每天最多1次，全量诊断每周最多1次（7天冷却），自动巩固每天最多2次。

## 四、诊断机制

### 4.1 诊断的三层架构

第一层：快速诊断（5分钟）——知识点覆盖率检查、错题/遗忘模式识别、注意力/效率趋势。

第二层：增量诊断（30分钟）——新增知识点的掌握度评估、新旧知识连接强度检测、学习方法有效性分析。

第三层：全量诊断（2小时，每周/月）——完整知识图谱分析、学习路径效率评估、方法组合效果回溯、长期趋势预测。

### 4.2 薄弱点识别算法

步骤：
1. 知识图谱构建：从捕获资料提取概念节点及其关系边
2. 掌握度标注：每个节点的掌握度是最后一次巩固时间、巩固次数和测试正确率的函数
3. 薄弱点识别：孤立节点（掌握度低于40）为严重薄弱，连接稀疏节点（关联边少于3）为中度薄弱，频繁出错节点（正确率低于70%）为值得关注
4. 排序后生成诊断报告

### 4.3 诊断报告结构

诊断报告包含概览信息（总知识点数、已掌握数、薄弱数、未探索数、总体健康分、趋势）、薄弱点清单（每个概念的名称、掌握度、严重程度、关联边数量、相关错误、改善建议）、推荐方案（巩固策略、优先级领域、预估投入、建议方法）以及元数据（数据来源、置信度、触发方式）。

### 4.4 诊断与学习科学的结合

参考知识库01_通用学习科学中的方法：主动回忆用于测试掌握度，费曼技巧用于评估理解深度，自我解释用于诊断思维模型缺陷，元认知用于评估学习策略有效性。

## 五、巩固机制

### 5.1 巩固输出的形式和用途

微巩固输出3-5个自测问题，用户耗时5分钟；增量巩固输出Anki卡片包加知识要点，耗时15-30分钟；全量巩固输出知识图谱加结构化笔记加复习计划，耗时1-2小时；长期记忆巩固输出间隔重复卡片更新，耗时取决于卡片数。

### 5.2 巩固执行的完整生命周期

触发后进入准备阶段（收集原料包括捕获资料、诊断报告、已有巩固记录），然后预处理（编码转换、去重、格式统一），核心处理包括知识点提取、理解层级判断、卡片笔记生成、难度标注，之后校验薄弱点覆盖率，后处理更新知识库索引和掌握度数据库，最终交付至用户选择的工具并登记反馈。

### 5.3 间隔重复集成

巩固系统的核心输出是间隔重复就绪的卡片包。每个知识点生成2-3张主动回忆式问题卡片。首次间隔根据掌握度动态设置：掌握度低于40时隔1天，40-60时隔3天，60-80时隔7天，高于80时隔14-30天。卡片通过AnkiConnect API或CSV导出推送至Anki。

## 六、与 01 通用学习科学的对接

### 6.1 直接引用的知识库内容

本系统的理论根基来自以下知识库文件：主动回忆（掌握度测试方法和卡片生成机制）、间隔重复（巩固间隔计算公式和策略时间规划）、练习测试与测试效应（错题模式识别）、费曼技巧（理解深度评估）、自我解释与精细提问（思维模型缺陷分析）、元认知与自我调节学习（诊断报告的自我评估校准）、错误复盘系统（错题归类和复盘）、Anki使用指南（工具对接实现）。

### 6.2 方法组合推荐

基于诊断结果，系统建议特定的学习方法组合：
- 概念理解浅层（仅记忆层面）：费曼技巧60%加主动回忆40%
- 概念间缺乏连接：思维导图加自我解释加交错练习
- 应用场景不熟：项目驱动加刻意练习加变式训练
- 遗忘率过高：间隔重复70%加主动回忆30%
- 错题模式重复：错误复盘加变式训练
- 学习动力下降：元认知反思加目标分解

## 七、数据结构设计

### 7.1 诊断记录

诊断记录包含diagnosisId、type（daily/weekly/monthly）、triggeredAt、triggerType（auto/manual）、state（总知识点数、已掌握数、薄弱数、未探索数、总体健康分、认知负荷、趋势）、weaknesses数组（每个元素包含concept、domain、masteryLevel、severity、symptoms、rootCauses、recommendedMethods、firstIdentified、consolidationCount）、recommendation（巩固策略、优先级领域、每日预估时间、持续天数、建议方法、待创建卡片数、待复习卡片数）、confidence和数据来源。

### 7.2 巩固记录

巩固记录包含consolidationId、strategy（incremental/full/micro/auto/longterm）、triggeredBy（关联诊断ID）、status（generated/delivered/completed/failed）、input（来源诊断ID、来源捕获ID、覆盖知识点数）、output（Anki牌组名、生成的卡片数、核心笔记摘要、概念关系图）、quality（薄弱点覆盖率、卡片质量评分、用户评分、是否需要人工审核）、timing（生成时间、交付时间、完成时间、预计时长）和version信息。

### 7.3 索引设计

主索引为diagnosisId，薄弱点索引为concept字段，时间索引为triggeredAt，状态索引为overallHealth，巩固策略索引为strategy，因果关系索引为triggeredBy。

## 八、总结与下一步

诊断巩固系统从 Claude Code 的 Compact 系统中汲取了核心设计模式，将 AI 对话中的上下文压缩策略成功映射到人类学习的知识巩固场景。五策略体系（全量/增量/自动/微/长期）覆盖了从每日5分钟快速回顾到月度深度整合的所有场景需求。
