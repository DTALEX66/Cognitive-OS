## 一、系统定位

### 1.1 主动训练系统在学习 OS 中的角色

主动训练系统是 A-Line Human Learning OS 的核心动力系统，承担着**「将被动输入转化为主动输出」**的关键职责。在人类学习的完整链路中，捕获（01）和诊断（02）解决了「输入什么」的问题，计划（03）解决了「如何安排」的问题，而主动训练系统解决的是「如何真正练会」的问题。

整个 A-Line 的学习流水线中，训练系统处于承上启下的枢纽位置：

`
捕获 → 诊断 → 计划 → 训练 → 记忆 → 反馈 → 迭代
                         ↑
               (主动训练是计划的执行引擎)
`

- 捕获（01）采集学习资料 → 诊断（02）识别薄弱点 → 计划（03）制定学习路线 → 主动训练（07）执行具体练习任务 → 记忆（04）巩固长期记忆 → 反馈（05）追踪效果形成闭环

主动训练系统区别于传统的「被动学习」（如反复阅读、划重点），其核心理念来源于 CC Hooks 系统的**自动化触发模式**——不再依赖用户的主观意愿去「决定何时练习」，而是通过预先定义的条件和 Hook 链，在恰当的时机自动触达训练任务。

### 1.2 与 CC Hooks 系统的渊源

本系统的设计灵感来自 Claude Code 的 Hooks 系统。CC 的 Hook 机制是一个事件驱动的中间件架构——在工具执行的不同生命周期阶段（执行前、执行后、失败、权限拒绝）插入自定义逻辑。迁移到学习系统语境后，这个「生命周期触发」模式完美对应了学习过程中的各个关键时刻：

| CC Hooks 概念 | A 线迁移 | 主动训练映射 |
|--------------|---------|------------|
| Pre-Tool Hooks | 学习前触发 | 材料准备 → 目标设定 → 热身练习 |
| Tool 执行 | 学习过程 | 主动练习/测试/应用 |
| Post-Tool Hooks | 学习后触发 | 摘要生成 → 记忆更新 → 后续推荐 |
| Post-Tool Failure Hooks | 学习失败处理 | 错题记录 → 降级方案 → 补救推荐 |
| Permission-Denied Hooks | 规则违反处理 | 超时提醒 → 注意力干预 → 模式调整 |

### 1.3 主动训练的核心原则

主动训练系统的运作建立在三条核心原则之上：

**原则一：触发自动化** —— 训练不应依赖意志力。通过智能触发机制，在认知状态最佳的时机自动推送训练任务，降低决策成本和行动门槛。

**原则二：反馈即时化** —— 每次训练必须产生可量化的反馈。没有反馈的练习只是重复，没有修正的反馈只是评判。即时反馈是刻意练习区别于普通练习的关键。

**原则三：渐进结构化** —— 训练任务按照「最近发展区」理论，从当前水平出发，逐步增加难度和复杂度，确保始终处于最佳挑战区间。

## 二、Hook 链设计

### 2.1 五阶段 Hook 链架构

主动训练系统采用与 CC Hooks 一脉相承的五阶段生命周期架构。每个学习事件从触发到完成，经历完整 Hook 链：

`
                            +--------------------+
                            |  事件触发条件       |
                            | (定时/条件/手动)    |
                            +---------+----------+
                                      |
           +------------------------------------------------------+
           |       Phase 1: Pre-Train Hooks                       |
           |   材料准备 . 目标设定 . 热身                          |
           +---------------------------+--------------------------+
                                       |
           +------------------------------------------------------+
           |       Phase 2: Active Training                       |
           |   练习执行 . 即时反馈 . 调整                          |
           +---------------------------+--------------------------+
                                       |
                             +---------------------+
                             |  检查结果状态        |
                             +--+--------+--------+--+
                                |        |           |
                        +-------+  +--------+  +-----------+
                        | 成功   |  | 失败   |  | 规则违反  |
                        +---+----+  +----+---+  +-----+-----+
                            |           |            |
                    +-------------+ +---------+ +---------------+
                    | Phase 3:   | | Phase 4:| | Phase 5:     |
                    | Post-Train | | Failure | | Rule-Violation|
                    | Hooks      | | Hooks   | | Hooks        |
                    +-------------+ +---------+ +---------------+
`

### 2.2 Phase 1：学习前触发（Pre-Train Hooks）

在正式训练开始之前触发的预处理逻辑。对应 CC 的 executePreToolHooks，但迁移到学习语境后，职责更为丰富。

`python
class PreTrainHooks:
    def execute(self, session):
        # Hook 1: 材料准备
        self.prepare_materials(session.subject)
        # 从知识库提取相关学习资料，加载上次训练进度
        
        # Hook 2: 目标设定
        self.set_objectives(session)
        # 根据 FSRS 数据确定本次训练目标，设定难度分级
        
        # Hook 3: 状态检测
        self.check_readiness(session)
        # 检测当前认知负荷，确认是否有足够时间完成训练
        
        # Hook 4: 热身练习
        self.warm_up(session)
        # 快速回顾上次训练的核心概念，激活相关记忆模式
`

**Hook 结果类型**：

| 返回结果 | 含义 | 处理方式 |
|---------|------|---------|
| blocking（阻塞） | 条件不满足 | 中断训练，推送提示说明原因 |
| pass（通过） | 条件满足 | 进入 Phase 2 正式训练 |
| warning（警告） | 存在风险 | 继续执行但记录警告信息 |

**典型阻塞场景**：
- 检测到学习者疲劳指数过高 → 建议休息，推迟训练
- 检测到前置知识点未掌握 → 推送补学通知
- 预估训练时间不足 → 缩短本次训练深度

### 2.3 Phase 2：主动训练执行

实际的训练过程不是盲目的练习，而是一个**检测—执行—反馈—调整**的闭环：

`python
class ActiveTrainingPhase:
    def execute(self, session):
        # 难度校准
        difficulty = self.calibrate_difficulty(session.profile)
        # 基于 FSRS 参数和历史表现自动校准
        
        # 任务分配
        task = self.assign_task(session, difficulty)
        
        # 执行与实时监控
        while not session.is_complete:
            result = self.perform_step(task)
            
            # 实时反馈
            feedback = self.generate_immediate_feedback(result)
            self.present_feedback(feedback)
            
            # 动态调整
            if self.needs_adjustment(result, difficulty):
                difficulty = self.recalibrate(result)
                task = self.adjust_task(session, difficulty)
            
            # 超时保护
            if self.is_timeout(session):
                self.enforce_break(session)
                break
`

### 2.4 Phase 3：学习后触发（Post-Train Hooks）

训练完成后执行的后处理逻辑，对应 CC 的 executePostToolHooks：

`python
class PostTrainHooks:
    def execute(self, session, result):
        # Hook 1: 生成学习摘要
        summary = self.generate_summary(session, result)
        # 提炼本次训练的关键收获，标记难点和易错点
        
        # Hook 2: 更新记忆参数
        self.update_memory_params(result)
        # 调整 FSRS 参数，更新间隔重复调度
        
        # Hook 3: 更新学习画像
        self.update_learning_profile(result)
        # 更新子系统 06 的学习画像数据，记录技能掌握度变化
        
        # Hook 4: 推送后续推荐
        recommendations = self.generate_recommendations(result)
        # 根据本次表现推荐下一步训练，建议交错练习的下一主题
`

### 2.5 Phase 4：学习失败处理（Failure Hooks）

当训练过程中出现失败或错误时触发。对应 CC 的 executePostToolUseFailureHooks：

`python
class FailureHooks:
    def execute(self, session, error):
        if error.type == ErrorType.CONCEPT_MISMATCH:
            self.record_misconception(error)
            self.recommend_remediation(error.concept)
            # 记录概念混淆，推荐基础补学
            
        elif error.type == ErrorType.DIFFICULTY_TOO_HIGH:
            self.lower_difficulty(session)
            self.schedule_retry(session, delay="15min")
            # 降低难度等级，休息后重试
            
        elif error.type == ErrorType.ATTENTION_LOST:
            self.trigger_refocus(session)
            self.adjust_session_length(session)
            # 触发重新聚焦提醒，缩短训练时长
        
        elif error.type == ErrorType.INSUFFICIENT_TIME:
            self.save_checkpoint(session)
            self.schedule_continuation(session)
            # 保存训练进度，安排在下次继续
`

### 2.6 Phase 5：规则违反处理（Rule-Violation Hooks）

当学习者违反预先设定的学习规则时触发。对应 CC 的 executePermissionDeniedHooks：

`python
class RuleViolationHooks:
    def execute(self, session, violation):
        if violation.type == ViolationType.TIME_LIMIT_EXCEEDED:
            self.enforce_break(session, minutes=10)
            # 强制休息 10 分钟
            
        elif violation.type == ViolationType.CHEAT_DETECTED:
            self.invalidate_current_session(session)
            self.trigger_honor_review()
            # 作废本次训练成绩，触发诚信审查
            
        elif violation.type == ViolationType.TOPIC_DEVIATION:
            self.redirect_to_topic(session)
            # 引导回原定主题
            
        elif violation.type == ViolationType.FATIGUE_WARNING:
            self.suggest_mode_switch("review")
            # 建议切换为复习模式
`

## 三、训练任务类型

### 3.1 任务分类体系

主动训练系统支持六种核心训练任务类型，每一种对应不同的学习目标和认知需求：

| 任务类型 | 学习目标 | 适用场景 | 推荐时长 |
|---------|---------|---------|:-------:|
| 记忆提取 | 巩固长期记忆 | 新学知识复习 | 10-20min |
| 概念辨析 | 区分相似概念 | 易混淆知识点 | 15-25min |
| 应用迁移 | 将知识应用于新场景 | 技能训练 | 20-40min |
| 错题修复 | 纠正错误理解 | 诊断后的修复 | 15-30min |
| 综合训练 | 多项能力联合训练 | 考前/阶段总结 | 30-60min |
| 探索学习 | 知识边界拓展 | 进阶/拓展 | 20-30min |

### 3.2 记忆提取任务

基于主动回忆（Active Recall）和测试效应（Testing Effect）的训练类型。

`python
class MemoryRetrievalTask(TrainingTask):
    def design(self, subject, profile):
        # 确定提取内容：间隔接近遗忘曲线的项目优先
        items = self.select_items_for_retrieval(
            subject=subject,
            review_history=profile.review_history,
            fsrs_params=profile.fsrs_parameters
        )
        
        for item in items:
            task_item = RetrievalItem(
                prompt=self.design_prompt(item),
                expected_answer=item.key_concept,
                hint_level=self.calculate_hint_level(item, profile),
                max_attempts=3,
                feedback_strategy="immediate_correction"
            )
            self.items.append(task_item)
        
        # 从易到难的难度梯度
        self.difficulty_progression = [0.3, 0.5, 0.7, 0.9]
`

### 3.3 概念辨析任务

针对易混淆概念的对比训练，基于交错练习（Interleaved Practice）原理：

`python
class ConceptDiscriminationTask(TrainingTask):
    def design(self, concepts):
        # 生成对比对
        pairs = self.generate_confusion_pairs(concepts)
        
        for pair in pairs:
            # 交错呈现
            self.add_interleaved_sequence(pair)
            
            # 在每个切换点加入判断
            self.add_discrimination_check(
                concept_a=pair[0],
                concept_b=pair[1],
                question="以下描述属于哪个概念？"
            )
`

**关键设计要素**：
1. 超维特征提取：训练识别概念的独有特征
2. 反例暴露：刻意呈现边界模糊的案例
3. 比较框架：提供对比表格，要求填充差异点

### 3.4 应用迁移任务

将所学知识应用到新情境的训练，基于变式训练（Varied Practice）原理：

`python
class ApplicationTransferTask(TrainingTask):
    def design(self, concept):
        # 表面变式：改场景不改结构
        self.add_variant(VariantType.SURFACE, 
            context_change=True, structure_change=False)
        # 结构变式：改结构不改核心
        self.add_variant(VariantType.STRUCTURAL,
            context_change=False, structure_change=True)
        # 极端变式：边界条件
        self.add_variant(VariantType.EXTREME, boundary_condition=True)
        # 跨域变式：映射到不同领域
        self.add_variant(VariantType.CROSS_DOMAIN, domain_change=True)
`

### 3.5 错题修复任务

根据诊断系统和反馈系统发现的知识漏洞，进行针对性修复：

`
诊断系统发现的薄弱点
       |
识别错误类型：概念混淆 / 记忆缺失 / 方法错误 / 理解偏差
       |
选择修复策略：
  - 概念混淆 -> 对比训练 + 特征提取
  - 记忆缺失 -> 间隔提取 + 关键词联想法
  - 方法错误 -> 样例学习 + 分步演练
  - 理解偏差 -> 费曼技巧 + 自我解释
       |
执行修复训练 + 验证测试
       |
确认修复后更新诊断画像
`

当多个子技能需要联合训练时，设计综合任务：

## 四、训练流水线

### 4.1 完整流水线架构

从触发到完成的完整训练流水线由 8 个阶段组成：

`
                       定时触发         条件触发         手动触发
                           |              |              |
                     +---------------------------------------------+
                     |     Stage 0: 触发条件评估                    |
                     |   (可用时间/认知状态/前置条件检查)            |
                     +---------------------+-----------------------+
                                           |
                     +---------------------------------------------+
                     |     Stage 1: 任务生成                        |
                     |   (选择类型/设计内容/校准难度)                |

                     +---------------------------------------------+

                     +---------------------------------------------+
                     |     Stage 3: 主动训练执行                    |
                     |   (检测 -> 执行 -> 反馈 -> 调整的迭代循环)   |
                     +---------------------+-----------------------+
                                           |
                     +---------------------------------------------+
                     |     Stage 4: 结果评估                        |
                     |   (成功率/时间/认知负荷/进步量)               |
                     +-----------+---------------------+-----------+
                                 |                     |
                        +--------+------+       +------+----------+
                        | 成功 -> 5a   |       | 失败 -> 5b     |
                        | Post-Train   |       | Failure Hooks  |
                        +-------+------+       +--------+--------+
                                |                       |
                     +---------------------------------------------+
                     |     Stage 6: 数据持久化                      |
                     |   (更新画像/更新FSRS/记录日志)                |
                     +---------------------+-----------------------+
                                           |
                     +---------------------------------------------+
                     |     Stage 7: 后续调度                        |
                     |   (安排下次训练/推送推荐/更新计划)            |
                     +---------------------------------------------+
`

### 4.2 触发条件评估（Stage 0）

`python
class TriggerEvaluator:
    def evaluate(self, candidates):
        for trigger in sorted(candidates, key=self.priority_score):
            # 条件一：时间窗口是否足够
            if not self.has_sufficient_time(trigger.estimated_duration):
                continue
            
            # 条件二：认知状态是否良好
            cognition = self.assess_cognitive_state()
            if cognition.fatigue_level > 0.7:
                self.suggest_rest_before_train(trigger)
                continue
            
            # 条件三：前置知识是否就绪
            if not self.check_prerequisites(trigger.subject):
                self.queue_remediation(trigger.subject)
                continue
            
            # 条件四：距离上次训练间隔是否合理
            if not self.check_training_interval(trigger.subject):
                continue
            
            return trigger
        
        return None  # 没有符合条件的触发
`

**触发优先级评分公式**：

`
Priority = w1 x Near_FSRS_forgetting + w2 x Exam_proximity + w3 x Motivation_signal
         - w4 x Fatigue_index - w5 x Schedule_conflict

默认权重: w1=0.35, w2=0.25, w3=0.20, w4=0.15, w5=0.05
`

### 4.3 训练调度算法

**时间-任务匹配规则**：

| 时间段 | 认知特点 | 推荐任务类型 |
|--------|---------|------------|
| 清晨（6:00-8:00） | 认知资源充足、干扰少 | 应用迁移、综合训练 |
| 上午（9:00-11:30） | 专注度高 | 概念辨析、错题修复 |
| 午后（14:00-16:00） | 中等专注、略疲劳 | 记忆提取（低难度） |
| 晚间（19:00-21:00） | 放松状态 | 探索学习、反思性训练 |
| 碎片时间（<15min） | 注意力分散 | 快速提取、打卡式复习 |

### 4.4 执行中的动态调整

训练不是一成不变的执行过程。系统在训练过程中持续监控四个关键指标：

`python
class DynamicTrainAdjuster:
    def monitor_and_adjust(self, session):
        while session.is_active:
            metrics = self.collect_live_metrics(session)
            
            # 难度自适应
            if metrics.accuracy > 0.9:
                session.increase_difficulty(factor=0.1)
            elif metrics.accuracy < 0.5:
                session.decrease_difficulty(factor=0.15)
                if metrics.consecutive_failures > 3:
                    self.suggest_topic_review(session)
            
            # 时间自适应
            if metrics.time_elapsed > session.expected_duration * 1.5:
                self.accelerate_pace(session)
            elif metrics.fatigue_index > 0.8:
                self.enforce_break(session)
            
            # 内容自适应
            if metrics.boredom_index > 0.7:
                self.switch_question_style(session)
                # 如选择题改为填空题，增加认知挑战
            
            # 模式自适应
            if metrics.frustration_index > 0.8:
                self.suggest_mode_switch("review")
                # 从专注模式切换到复习模式
`

## 五、与 09 知识库 04_应用与迁移的对接

### 5.1 对接架构概览

主动训练系统的训练方法设计深度依赖 09 知识库中「04_应用与迁移」模块的学习科学原理。两者通过一个标准化的**方法引用接口**实现对接：

`python
class KnowledgeBaseAdapter:
    def __init__(self):
        self.method_registry = {
            "deliberate_practice": DeliberatePracticeMethod(),
            "interleaved_practice": InterleavedPracticeMethod(),

class TrainingEffectivenessModel:
    def evaluate_session(self, session_data):
        report = EffectivenessReport()
        
        # 1. 即时效果：本次训练的表现变化
        report.immediate_gain = self.compute_immediate_gain(
            pre_test=session_data.pre_test_score,
            post_test=session_data.post_test_score
        )
        
        # 2. 保持效果：间隔测试的表现衰减率
        report.retention_rate = self.compute_retention_rate(
            immediate_post=session_data.post_test_score,
            delayed_test=session_data.delayed_test_score,
            delay_hours=session_data.delay_hours
        )
        
        # 3. 迁移效果：应用到新情境的能力
        report.transfer_score = self.compute_transfer_capability(
            familiar_accuracy=session_data.familiar_test_score,
            novel_accuracy=session_data.novel_test_score
        )
        
        # 4. 效率指标：单位时间的学习收益
        report.efficiency = self.compute_efficiency(
            total_gain=report.immediate_gain,
            time_spent=session_data.total_duration_minutes,
            cognitive_cost=session_data.cognitive_load
        )
        
        return report
`

### 6.6 持续优化闭环

训练系统的最终目标是从每次训练中学习，不断自我优化：

`
每次训练完成
      |
采集多维度数据
      |
分析效果指标：哪些训练设计有效？哪些失效？
      |
提炼模式规则：
  - 学习者在晚上 8 点后更适合记忆提取而非综合训练
  - 当挫败感 > 0.7 时，继续施压会导致放弃
  - 交错比例在 0.6 时效果最优
      |
更新训练引擎参数
  - 调整默认训练时长
  - 优化难度校准算法
  - 改进方法组合策略
      |
下一次训练在优化后的配置上运行
`

**优化目标函数**：

`
Opt(T) = maximize(Effectiveness(T) - Cost(T))

其中：
  Effectiveness = w1 x Retention + w2 x Transfer + w3 x Efficiency
  Cost = w4 x Fatigue + w5 x Time + w6 x Frustration
`

通过持续优化，训练系统不断逼近每个学习者的个性化最优训练策略。
