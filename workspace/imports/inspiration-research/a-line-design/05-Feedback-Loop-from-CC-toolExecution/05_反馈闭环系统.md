# 05 | 反馈闭环系统 — Feedback Loop System

## 一、系统定位：反馈闭环在 OS 中的角色

### 1.1 从 CC toolExecution 到学习反馈

A-Line Human Learning OS 的反馈闭环系统，脱胎于 Codex CLI 的 toolExecution 引擎（1,745 行核心代码）及其 PermissionContext 子系统（379 行）。在 Codex CLI 中，toolExecution 负责调度所有工具调用（文件读写、Shell 命令、代码执行），并通过 Hooks 机制在各个执行阶段插入拦截逻辑。PermissionContext 则提供细粒度的权限校验。

将这个机制迁移到学习领域，我们得到的是一个**完整的反馈闭环系统**——它不仅仅是对错判断，而是覆盖了学习行为的全生命周期：

- **执行前（Pre-Hook）**：检查前提条件、预估难度、激活相关知识
- **执行中（Execution）**：完成学习任务（做题、复习、实践）
- **执行后（Post-Hook）**：校验结果、分析表现、生成反馈
- **失败处理（Failure Hook）**：错误归因、分级响应、补救路径
- **权限阻断（Permission Denied Hook）**：条件不足时的降级或预处理

### 1.2 在 A-Line 架构中的位置

反馈闭环系统处于 A-Line 的**执行层与评估层交界处**：

```
输入层（Capture 子系统）
    ↓
诊断层（Diagnose 子系统）→ 定位当前知识状态
    ↓
规划层（Plan 子系统）→ 制定学习路径
    ↓
执行层（Execute）→ 学习行为发生
    ↓
★★★ 反馈闭环（Feedback Loop）★★★
    ↓
画像层（Profile 子系统）→ 更新学习者画像（06 子系统）
    ↓
知识层（Palace 子系统）→ 更新知识宫殿（04 子系统）
    ↓
反馈回流 → 影响下一轮规划
```

每一次学习行为的输出，经过反馈闭环处理后，产生三个方向的回流：

1. **向上反馈**：给规划层（03 子系统）——调整后续路径
2. **横向写入**：给知识宫殿（04 子系统）——更新知识点掌握度
3. **纵向积累**：给画像系统（06 子系统）——更新学习画像

### 1.3 设计原则

- **即时反馈**：学习行为完成后立即产生反馈，不延迟
- **分级处理**：不同类型的反馈走不同的处理流程
- **数据驱动**：所有反馈都结构化存储，支持统计和模式发现
- **自适应**：反馈阈值根据学习者水平和历史表现动态调整

## 二、4 种 Hook 映射 — CC Hook 机制 → 学习反馈流程

### 2.1 Hook 架构总览

CC toolExecution 定义了四种 Hook 类型，A-Line 将其完整映射为学习反馈的四大环节：

| CC Hook 类型 | 时机 | A-Line 映射 | 典型场景 |
|-------------|------|------------|---------|
| PreToolHook | 工具执行前 | **前置校验（Pre-Check）** | 检查前置知识是否掌握、预估题目难度是否合适 |
| PostToolHook | 工具执行后 | **结果校验（Post-Check）** | 判断答案是否正确、分析解题过程、记录时间 |
| FailureHook | 工具执行失败 | **错误处理（Error Handler）** | 记录错误信息、触发错题流程、推送补救材料 |
| PermissionDeniedHook | 权限不足 | **条件不足处理（Prerequisite Handler）** | 前置知识点未掌握时推送预习内容、跳过高难度题目 |

### 2.2 Pre-Check（前置校验）

**CC 原型：** PreToolHook 在执行前检查工具参数的有效性和安全性。

**A-Line 映射：** 在学习行为开始前执行一系列前置检查，避免无效学习或超出当前能力范围的任务。

```python
class PreCheckHook:
    def execute(self, task: LearningTask) -> PreCheckResult:
        checks = [
            self.check_prerequisites(task),      # 前置知识是否就绪
            self.check_difficulty(task),          # 难度是否匹配当前水平
            self.check_time_available(task),      # 时间是否充足
            self.check_energy_level(task),        # 学习状态是否合适
        ]
        failed = [c for c in checks if not c.passed]
        if failed:
            return PreCheckResult(
                passed=False,
                block_reasons=failed,
                suggestions=[c.remediation for c in failed]
            )
        return PreCheckResult(passed=True)
```

### 2.3 Post-Check（结果校验）

**CC 原型：** PostToolHook 在执行完成后收集结果、记录日志、触发后续操作。

**A-Line 映射：** 学习行为完成后，对结果进行全面校验——不仅是正确/错误的二元判断，还包括过程分析、时间效率、思考深度等多维评估。

```python
class PostCheckHook:
    def evaluate(self, task: LearningTask, answer: Answer) -> Evaluation:
        evaluation = Evaluation(
            task_id=task.id,
            is_correct=self.verify_correctness(answer, task.expected),
            time_spent=answer.time_spent,
            time_efficiency=self.calc_efficiency(answer.time_spent, task.difficulty),
            confidence_level=answer.confidence,
            thinking_depth=self.analyze_thinking(answer.process),
            hesitation_points=self.find_hesitations(answer.process),
        )
        evaluation.feedback = self.generate_feedback(evaluation)
        return evaluation
```

### 2.4 Error Handler（错误处理）

**CC 原型：** FailureHook 在工具调用失败时触发——记录错误堆栈、尝试降级方案、通知上级。

**A-Line 映射：** 答案错误时触发完整的错误处理流水线：捕获 → 分类 → 归因 → 推荐。

```python
class ErrorHandlerHook:
    def handle(self, task: LearningTask, wrong_answer: Answer) -> ErrorResolution:
        error = self.capture_error_detail(task, wrong_answer)
        error.category = self.classify_error(error)
        error.root_cause = self.attribute_root_cause(error)
        recommendations = self.recommend_remediation(error)
        self.write_to_error_notebook(error)
        return ErrorResolution(error=error, recommendations=recommendations)
```

### 2.5 Prerequisite Handler（条件不足处理）

**CC 原型：** PermissionDeniedHook 在权限不足时触发——提示缺失权限、推荐申请流程、降级功能。

**A-Line 映射：** 当学习者尝试学习还未准备好（前置知识不足）的内容时，触发降级处理。

```python
class PrerequisiteHandlerHook:
    def handle(self, task: LearningTask) -> Remediation:
        missing = palace.get_missing_prerequisites(task.topic)
        if missing:
            return Remediation(
                action="redirect",
                message=f"需要先掌握以下内容: {missing}",
                redirect_to=missing[0],
                estimated_prep_time=self.estimate_time(missing)
            )
```

## 三、错误反馈机制 — 学习错误的分级和处理流程

### 3.1 错误分级体系

参考 CC 的异常分级（Error/Warning/Info）和 04_错题本模板的错误类型分类，A-Line 定义了六级错误体系：

| 级别 | 名称 | 说明 | 处理策略 |
|-----|------|------|---------|
| P0 | 核心概念错误 | 该领域最重要的概念出错 | 立即停下手头任务，推送概念精讲 + 3道同类题 |
| P1 | 常见模式错误 | 大量学习者在此出错的模式 | 记录到高频错题集，安排专项练习 |
| P2 | 方法性错误 | 解题方法/思路选择错误 | 推送方法对比分析 + 思维导图 |
| P3 | 理解偏差错误 | 理解正确但方向偏离 | 标注理解偏差点，推送对比案例 |
| P4 | 计算/熟练度 | 计算失误、公式记混等 | 记录到熟练度追踪，适量练习即可 |
| P5 | 纯粗心错误 | 抄错数、看错条件 | 简要标记，提示注意力检查清单 |

### 3.2 错误处理流水线

```
学习行为完成
    ↓
[是否正确？]───是──→ 进入正向反馈机制
    ↓ 否
捕获错误详情（原题、错误答案、过程记录）
    ↓
错误分类器（基于错误特征自动分类 + 人工确认）
    ↓
P0 ──→ 紧急响应：概念精讲 → 即时练习 → 复查
P1 ──→ 专项练习：加入高频错题集 → 安排专题训练
P2 ──→ 方法对比：推送方法对比案例 → 思维导图
P3 ──→ 偏差修正：标注偏差点 → 推送对比材料
P4 ──→ 熟练度训练：加入常规练习队列
P5 ──→ 轻量标记：更新注意力提示清单
    ↓
错题记录 → 写入错题本模板（04_错题本模板格式）
    ↓
知识状态更新 → 更新知识宫殿的掌握度
    ↓
画像更新 → 更新学习画像的错误分布
```

### 3.3 自动分类器设计

错误分类器基于以下特征维度进行自动归类：

```python
class ErrorClassifier:
    def classify(self, error: ErrorDetail) -> ErrorCategory:
        features = {
            "knowledge_gap": self.detect_knowledge_gap(error),
            "method_choice": self.analyze_method_selection(error),
            "calculation": self.check_calculation_process(error),
            "attention": self.check_attention_error(error),
            "comprehension": self.analyze_comprehension_depth(error),
        }
        primary_feature = max(features, key=features.get)
        if primary_feature == "knowledge_gap" and features["knowledge_gap"] > 0.8:
            return ErrorCategory.P0
        elif primary_feature == "method_choice":
            return ErrorCategory.P2
        # ... 更多规则
```

### 3.4 错误恢复策略

不同级别的错误有不同的恢复策略：

- **P0 恢复**：推送 3-5 分钟的微课精讲 → 做 3 道同类题 → 正确率达到 80% 以上才继续
- **P1 恢复**：加入高频错题本 → 每周专项训练 15 分钟
- **P2 恢复**：推送方法对比思维导图 → 做 2 道需要用不同方法的对比练习
- **P3 恢复**：推送正反面对比案例 → 做 1 道类似题并写出思路分析
- **P4 恢复**：加入每日 5 分钟快速练习队列
- **P5 恢复**：无额外操作，仅在统计中标记

## 四、正向反馈机制 — 正确完成的强化策略

### 4.1 正确完成的分级响应

正向反馈不仅仅是"做对了，很好"，而是根据完成质量进行分级强化：

**评估维度：**
- 正确性：完全正确 / 部分正确 / 思路正确但结果偏差
- 效率：完成时间 / 预估时间比值
- 自信度：学习者自评信心（1-5）
- 过程完整性：是否展现了完整推理过程
- 迁移度：是否主动关联到其他知识点

**正向反馈分级：**
- **S级（完美掌握）**：正确 + 高效 + 高自信 + 完整过程 + 有关联 → 标记为已掌握，降低复习频率，推荐进阶内容
- **A级（良好掌握）**：正确 + 中等效率 + 完整过程 → 保持正常复习间隔，标记为稳定掌握
- **B级（基本掌握）**：正确但效率低 / 过程不完整 → 建议增加熟练度训练，保持较短复习间隔
- **C级（勉强正确）**：部分正确 / 思路有偏差但凑巧正确 → 按 P3 级别处理，需要深入检查理解偏差

### 4.2 间隔复习的反馈调整

基于正确性动态调整间隔重复（参考 FSRS 算法思想）：

- 连续 3 次正确 → 复习间隔乘以 1.5
- 混合表现（对错交替）→ 复习间隔保持不变
- 连续 2 次错误 → 复习间隔乘以 0.5，触发出错流程

### 5.3 模式发现引擎

积累足够的反馈数据后，系统可以自动发现学习模式：

```python
class PatternDiscovery:
    def discover_patterns(self, history: List[FeedbackRecord]) -> List[LearningPattern]:
        patterns = []
        # 时间模式：特定时间段的表现差异
        time_pattern = self.analyze_time_patterns(history)
        if time_pattern:
            patterns.append(time_pattern)
        # 类型聚集模式：某类错误是否集中出现
        cluster_pattern = self.detect_error_clusters(history)
        if cluster_pattern:
            patterns.append(cluster_pattern)
        # 链式错误模式：某个知识点出错后，后续相关知识点连锁出错
        chain_pattern = self.detect_error_chains(history)
        if chain_pattern:
            patterns.append(chain_pattern)
        # 复习衰减模式：刚复习完表现好，之后快速下降
        decay_pattern = self.detect_review_decay(history)
        if decay_pattern:
            patterns.append(decay_pattern)
        return patterns
```

### 6.2 对接知识宫殿子系统（04 子系统）

反馈结果直接影响知识宫殿中各知识点的掌握度指标：

```python
class PalaceUpdater:
    def update_from_feedback(self, feedback: FeedbackRecord):
        topic = feedback.topic
        if feedback.correct and feedback.evaluation_grade in ('S', 'A'):
            palace.boost_mastery(topic, amount=0.1)
            palace.extend_interval(topic, multiplier=1.5)
        elif feedback.correct and feedback.evaluation_grade == 'B':
            palace.boost_mastery(topic, amount=0.05)
            palace.extend_interval(topic, multiplier=1.2)
        else:
            palace.reduce_mastery(topic, amount=self.calc_penalty(feedback))
            palace.shrink_interval(topic, multiplier=0.5)
            palace.mark_for_review(topic, priority=feedback.error_category)
```

## 七、反馈到路径的联动 — 反馈结果如何影响后续学习路径

### 7.1 路径调整引擎

反馈闭环与路径规划（03 子系统）之间形成了一个紧密的联动循环：

```python
class PathAdjuster:
    def adjust(self, current_plan: LearningPlan, recent_feedback: List[FeedbackRecord]) -> AdjustedPlan:
        adjustments = []
        error_patterns = self.analyze_error_patterns(recent_feedback)

        for pattern in error_patterns:
            if pattern.type == "knowledge_gap_cluster":
                adjustments.append(PathAdjustment(
                    type="insert_remediation",
                    topic=pattern.cluster_topic,
                    position="next",
                    reason=f"发现知识盲区: {pattern.cluster_topic}"
                ))
            elif pattern.type == "chain_failures":
                adjustments.append(PathAdjustment(
                    type="slow_down",
                    multiplier=0.7,
                    reason="链式错误表明基础不够扎实"
                ))
            elif pattern.type == "mastery_achieved":
                adjustments.append(PathAdjustment(
                    type="advance",
                    target=self.find_next_stage(pattern.topic),
                    reason=f"{pattern.topic} 已完全掌握"
                ))

        new_plan = self.apply_adjustments(current_plan, adjustments)
        return AdjustedPlan(original=current_plan, adjusted=new_plan, adjustments=adjustments)
```

### 7.2 路径联动规则

| 反馈模式 | 路径调整 | 触发条件 |
|---------|---------|---------|
| 连续 3 次正确（S级） | 跳过当前难度的下一题，直接升级 | 同一知识点的连续 S 级评估 |
| 正确率低于 60% | 暂停推进新内容，增加复习量 50% | 最近 7 天的综合正确率 |
| 同一知识点连续 2 次 P0 错误 | 退回上一级知识点的复习 | 同一知识点的连续错误 |
| 某类错误占比大于 40% | 插入针对该错误类型的专题训练 | 最近 30 天错误的类型分布 |
| 所有知识点掌握度大于 85% | 进入下一章节/主题 | 知识宫殿的整体掌握度 |
| 学习效率持续下降 | 建议休息或更换学习方式 | 连续 3 天的效率下降趋势 |

### 7.3 自适应难度调整

反馈系统会动态调整后续任务的难度，使学习始终处于最近发展区：

```python
def calculate_next_difficulty(history: List[FeedbackRecord]) -> float:
    recent = history[-20:]
    accuracy = sum(1 for r in recent if r.correct) / len(recent)
    if accuracy > 0.85:
        return min(1.0, current_difficulty + 0.15)
    elif accuracy < 0.6:
        return max(0.1, current_difficulty - 0.15)
    else:
        return current_difficulty
```

### 7.4 完整的反馈-路径循环

```
[学习任务开始]
    ↓
Pre-Check（前置校验）
    ↓ 通过
[执行学习任务]
    ↓
Post-Check（结果校验）
    ↓
正确 ──→ 正向强化 → 更新知识宫殿掌握度 → 反馈数据积累
    ↓                                              ↓
错误 ──→ 错误分级 → 错题本写入 → 补救执行 ──→ 反馈数据积累
                                                    ↓
                                          [模式发现引擎]
                                                    ↓
                                        路径调整引擎 ←→ 画像系统
                                                    ↓
                                          [下一轮学习任务]
```

这个闭环确保每一次学习行为都不是孤立的——它的反馈会被记录、分析、并用于优化后续的所有学习活动，形成一个持续自我进化的学习系统。
